Mejora el diseño de interfaces en Figma de la plataforma web GreenScript Exchange (GSE), que conecta recicladores con centros de acopio para la compra y venta de botellas plásticas reciclables.
Mantén el diseño actual como base y agrega las pantallas, formularios y flujos de navegación necesarios para que el aplicativo sea funcional y coherente como Producto Mínimo Viable (PMV).

El diseño debe ser claro, intuitivo y nivel universitario, sin agregar funcionalidades complejas innecesarias.

Contexto actual del diseño:

Landing page con descripción del proyecto.

Botones: “Comenzar ahora” y “Conocer más”.

Marketplace con ofertas visibles.

Registro con selección de rol (Reciclador / Centro de Acopio).

Inicio de sesión.

Validación básica de campos.

Dashboard básico por rol.

Reciclador puede publicar.

Ambos roles tienen historial básico.

Mejoras que deben agregarse:
1️⃣ Pantallas necesarias

Dashboard del Reciclador con resumen (botellas vendidas, metas alcanzadas).

Dashboard del Centro de Acopio con resumen (botellas compradas, total transacciones).

Pantalla de detalle de publicación.

Pantalla de confirmación de compra.

Pantalla de reporte de impacto personal.

Pantalla de perfil editable.

Estados vacíos (sin publicaciones, sin historial).

Pantalla de confirmación exitosa tras publicar o comprar.

2️⃣ Formularios necesarios

Registro (nombre, correo, contraseña, rol).

Publicar botellas (tipo, cantidad, precio, condición, descripción).

Confirmación de compra (resumen + botón confirmar).

Edición de perfil (nombre, teléfono, dirección si es centro de acopio).

Agregar validaciones visuales:

Campos obligatorios.

Error de formato en correo.

Mensajes de confirmación.

3️⃣ Flujos de navegación que deben verse claros

Registro → Login → Dashboard según rol.

Reciclador publica → aparece en Marketplace.

Centro de acopio selecciona oferta → confirma compra → se guarda en historial.

Usuario entra a perfil → puede editar datos.

Desde historial → puede ver detalle de cada transacción.
